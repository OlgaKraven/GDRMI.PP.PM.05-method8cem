    Задание 2. Технические задания для художника

    Внимание! Данное задание выполняется на основе вашего игрового проекта.

    Цель задания

    Составить полный пакет технических заданий (ТЗ) для создания визуальных элементов игры: персонажей, локаций, интерфейса и ассетов, включая промпты для нейросетевой генерации.

    Техническое задание (ТЗ) — документ, по которому художник (или вы сами) создаёт визуальный элемент. Хорошее ТЗ позволяет получить именно то, что нужно, и избежать многократных переделок.

    Правило для работы: в основном тексте размещается по одному демонстрационному ТЗ каждого вида. Остальные материалы — в Приложение.

    2.1. ТЗ на главного персонажа

    Заполните все поля. Пустых строк быть не должно — если параметр не определён, укажите TBD и поясните, от чего зависит решение.

    Параметр Ваше описание
    Имя / обозначение персонажа Король Артур (Arthur)
    Роль в игре Главный герой, протагонист
    Функция в геймплее (управляемый / NPC / противник / союзник) Управляемый игроком
    Пол, возраст, комплекция Мужской, ~25 лет, атлетичное телосложение, широкие плечи
    Общий силуэт и пропорции Высокий (1.85 м), прямая осанка; силуэт читается по короне-шлему и плащу
    Лицо: черты, выражение, взгляд Резкие черты, короткая щетина, решительный взгляд, лёгкая усталость в глазах
    Костюм / снаряжение / аксессуары (по частям тела) Голова: шлем-корона с гравировкой дракона. Торс: золочёная кольчуга. Руки: наручи с рунами. Ноги: кожаные поножи. Спина: тёмно-синий плащ с гербом Камелота. Пояс: пустые ножны Экскалибура (слабое свечение)
    Доминирующие цвета (ссылка на палитру из Задания 1.2) Золото #C0A060 (броня), тёмно-синий #4A3F6B (плащ), серебро #C0C0C0 (шлем, наручи)
    Акцентные детали (что выделяет среди других) Пустые светящиеся ножны — визуальный нарратив: меч потерян, но место для него сохранено
    Характер: манера держаться, осанка, движения Держится прямо, движения уверенные, но не торопливые; idle — лёгкий наклон вперёд, как у готового к действию воина
    Психологический портрет (мотивация, внутренний конфликт) Движим чувством долга и стыда за потерю Экскалибура. Внутренний конфликт: сомневается в своём праве называться королём без меча
    Анимационные состояния idle (дыхание + колыхание плаща), walk, run, attack_light (J), attack_heavy (K), block (Shift), hurt, death (dissolve-эффект), interact (наклон вперёд + рука)
    Тип ассета ☑ спрайт-лист ☐ 3D-модель ☐ spine-анимация
    Разрешение спрайта / количество полигонов 128×128 px на кадр, PNG; итоговый спрайт-лист 1024×512 px (8 столбцов × 4 ряда)
    Формат файла на выходе .png (спрайт-лист) + .png (отдельные кадры для отладки)
    Особые пометки (маски, вариации, сезонные скины) Вариация: Артур с Экскалибуром (разблокируется после сбора всех 5 осколков) — отдельный спрайт-лист

    2.2. ТЗ на ключевую локацию

    Опишите одну ключевую игровую локацию — стартовую зону, главный хаб или первый уровень. Это локация, которая войдёт в вертикальный срез.

    Параметр Ваше описание
    Название локации Руины Камелота (Camelot Ruins)
    Тип локации (подземелье, город, лес, космическая станция и т.п.) Разрушенный средневековый замок, смешанная локация: внутренние залы + подземелье
    Роль в игре (стартовая зона / переход / боссовая / хаб) Стартовая зона — туториал + первый мини-босс
    Атмосфера и настроение Меланхолия и тяжесть утраты: бывшее величие теперь в руинах. Холодный лунный свет, мерцающие руны — следы проклятия Морганы
    Масштаб: размер, сколько экранов / комнат 4 комнаты, горизонтальный скроллинг; каждая комната ~3–4 ширины экрана
    Основные зоны и ключевые объекты 1) Тронный зал (старт); 2) Рунный коридор (загадка с кнопками); 3) Зал рыцарей (бой); 4) Рунный алтарь (NPC Мерлин + точка сохранения)
    Цветовая гамма (ссылка на палитру из Задания 1.2) #1A1A2E фон, #C0A060 золотые детали, #50C878 руны, #8B2FC9 следы магии Морганы
    Освещение: тип, источники света Ночное + внутреннее. Global Light 2D (синий, intensity 0.15). Point Light 2D: факелы (оранжевый), руны (зелёный), следы проклятия (фиолетовый)
    Интерактивные объекты Рунные кнопки на полу (открывают барьеры), Мерлин (NPC-диалог), Рунный алтарь (точка сохранения + способность), осколок Экскалибура
    Препятствия и враги Проклятые рыцари (3 шт.), падающие обломки (триггерные зоны), рунные барьеры
    Музыкальная тема / звуковое оформление Медленный оркестр: виолончель + хор (меланхолия). Ambient: далёкий ветер, капли воды, треск факелов. Источник: Tabletop Audio — "Camelot"
    Требования к тайлсету / Prefabs Tilemap 16×16 px: каменные стены, полы, обломки колонн, сломанные витражи; отдельные Prefabs: факел (Point Light2D + Particle), рунный барьер, кнопка-активатор
    Требования к Collider-у и физике CompositeCollider2D для tilemap. Отдельные BoxCollider2D / PolygonCollider2D для обломков. Trigger-зоны для барьеров и событий
    Требования к производительности (Draw Calls, Batching) Макс. 100 Draw Calls; спрайты объединить в Sprite Atlas; фоны — Static Batching
    Формат финальных ассетов PNG, тайлсет 16×16 px; отдельные элементы (факелы, декор) 32×32–64×64 px

    2.3. ТЗ на интерфейс: игровой HUD

    HUD (Heads-Up Display) — постоянно видимая информация на экране во время игры: здоровье, очки, инвентарь, мини-карта.

    Параметр Ваше описание
    Название экрана / элемента Игровой HUD
    Какую информацию отображает Здоровье Артура (сердца), шкала выносливости (stamina), счётчик осколков Экскалибура, активная способность (иконка), мини-карта (угол)
    Расположение каждого элемента Верх-лево: 5 сердец (здоровье). Верх-право: счётчик осколков [X/5] + мини-карта. Низ-центр: шкала выносливости (горизонтальная полоса). Низ-право: иконка активной способности
    Шрифт: название / гарнитура, кегль, начертание "MedievalSharp" (Google Fonts), 18 pt для значений счётчиков, 14 pt для подписей, Bold
    Цветовые акценты (ссылка на палитру) Рамки и разделители: #D4AF37 (золото). Текст: #F5F0E8 (тёплый белый). Сердца: #CC2200 (полные) / #4A1A1A (пустые)
    Иконки: стиль, размер в пикселях Плоские с золотым контуром, 32×32 px; осколки — 24×24 px
    Полоска здоровья: тип ☐ цифры ☐ полоска ☑ сердца ☐ другое
    Состояния элементов (нормальное / предупреждение / критическое) Нормальное: белые сердца. Предупреждение (≤ 2 сердец): медленная пульсация красным. Критическое (1 сердце): частая пульсация + красная виньетка по краям экрана
    Анимации HUD Получение урона: красная вспышка на 0.2 сек. Сбор осколка: иконка счётчика крутится + звонок. Использование выносливости: полоса мигает оранжевым при опустошении
    Адаптация под разрешения Основное: 1920×1080 (16:9). Дополнительно: 1280×720 — элементы масштабируются пропорционально через Canvas Scaler
    Canvas mode в Unity ☑ Screen Space Overlay ☐ Screen Space Camera ☐ World Space
    Шрифт в Unity ☑ TextMeshPro ☐ Legacy Text

    Схема расположения элементов HUD:

    ┌─────────────────────────────────────────────────────┐
│ ♥ ♥ ♥ ♥ ♥          [мини-карта]   [осколки: 1/5]  │
│                                                     │
│                                                     │
│                  (игровая сцена)                    │
│                                                     │
│            [════════════════] stamina               │
│                                          [⚡ ability]│
└─────────────────────────────────────────────────────┘

    ТЗ на главное меню (дополнительно)

    Элемент главного меню Описание / расположение / поведение
    Фон / задний план Анимированный: силуэт Камелота в тумане, медленно опадающие угли/листья (Particle System). Цвет: тёмно-синий #1A1A2E с градиентом в #0D0D1A снизу
    Логотип / название игры «Приключение короля Артура» в шрифте MedievalSharp, золотое свечение Bloom. Центр-верх, под короной с лучом света (статичная иллюстрация)
    Кнопка «Новая игра» Центр экрана, золотая рамка, белый текст. При активной сохранёнке — заменяется надписью «Продолжить»
    Кнопка «Продолжить» Под «Новой игрой», неактивна (#666666) при отсутствии сохранения
    Кнопка «Настройки» Под «Продолжить» — шестерёнка-иконка + текст
    Кнопка «Выход» Нижняя кнопка; в WebGL-версии скрыта
    Анимация / фоновая музыка Fade-in при входе (0.5 сек). Музыка: главная тема (оркестр, виолончель, хор) — loop
    Состояния кнопок normal: золотая рамка, белый текст. hover: рамка ярче #FFD700, масштаб ×1.05, звук «click». pressed: затемнение rgba(0,0,0,0.3). disabled: рамка серая, текст #888888

    2.4. ТЗ на ключевой ассет

    Опишите один ключевой игровой предмет или объект, важный для геймплея (оружие, ключ-карта, сундук, платформа, дверь, расходник и т.п.).

    Параметр Ваше описание
    Название предмета / объекта Осколок Экскалибура (Excalibur Shard)
    Категория ☐ оружие ☑ расходник (ключевой предмет) ☐ ключ ☐ декор ☐ платформа ☐ ловушка ☐ другое
    Функция в геймплее Сбор всех 5 осколков восстанавливает Экскалибур и разблокирует финальную способность. Каждый осколок при подборе открывает одну из пяти способностей Артура
    Визуальные характеристики: форма, размер, материал Фрагмент лезвия меча с зазубренными краями. В игровом пространстве ~40×20 px. Материал: светящееся золото с голубоватым свечением (как замороженный металл)
    Цветовая гамма Основной: #D4AF37 (золото). Свечение: #E8F4FF (холодный белый). Зазубрины: #A07840 (тёмное золото)
    Анимации ☐ статичный ☑ idle-анимация (медленное вращение 360° за 3 сек + пульсация свечения) ☑ при использовании (вспышка частиц + масштаб → 0 при подборе) ☐ разрушение
    Масштаб относительно персонажа ~1/5 высоты Артура (небольшой, но заметный благодаря свечению)
    Тип коллайдера ☐ Box ☑ Circle ☐ Polygon
    Технические параметры 64×64 px спрайт, 3-кадровая idle-анимация (вращение). Particle System (6–8 частиц, цвет #E8F4FF, lifetime 0.5 сек)
    Формат файла .png (спрайт-лист 192×64 px, 3 кадра)
    Вариации (состояния: открыто/закрыто, целое/сломанное) 5 визуальных вариантов осколков: кончик лезвия, гарда, середина клинка, рукоять, навершие. Каждый уникален по форме, но единый стиль
